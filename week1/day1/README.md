# Week 1 - Day 1: First Steps with Python and LLMs

## Overview
This project introduces the fundamentals of building a simple Python application that interacts with a Large Language Model (LLM). It is part of the AI Engineer Bootcamp and focuses on:

- writing a basic Python script
- loading environment variables securely
- calling the Groq API from Python
- printing a model-generated response

## Project Purpose
Day 1 is designed as a beginner-friendly introduction to working with AI APIs. The project demonstrates how to connect a Python application to an LLM service and retrieve a response from a prompt.

## Files Included
- `main.py` — a simple starter script that prints a basic greeting
- `hello_llm.py` — a script that sends a prompt to the Groq LLM and displays the response
- `pyproject.toml` — project metadata and Python dependencies

## Technologies Used
- Python 3.13+
- Groq API
- python-dotenv

## Prerequisites
Before running the project, make sure you have:

1. Python installed on your machine
2. A Groq API key
3. An environment file containing your API key

## Setup Instructions
1. Create and activate a virtual environment:
   ```bash
   python -m venv .venv
   source .venv/bin/activate
   ```
   On Windows PowerShell:
   ```powershell
   .\.venv\Scripts\Activate.ps1
   ```

2. Install the required dependencies:
   ```bash
   pip install groq python-dotenv
   ```

3. Create a file named `.env` in the project folder and add your Groq API key:
   ```env
   GROQ_API_KEY=your_api_key_here
   ```

   If you choose a different filename or location for your environment file, update the path in `hello_llm.py` accordingly.

## Running the Project
### 1. Run the basic Python script
```bash
python main.py
```

### 2. Run the LLM example
```bash
python hello_llm.py
```

This will send a sample prompt to the Groq model and print the generated answer in the terminal.

## Expected Output
- `main.py` prints a simple greeting message.
- `hello_llm.py` prints a response generated by the configured LLM.

## Notes
- Keep your API key private and do not commit it to version control.
- The project is intentionally simple and serves as an introduction to integrating AI services into Python applications.

## Summary
This Day 1 exercise provides a practical first look at connecting Python with an LLM using a real API. It is a strong foundation for more advanced AI-powered applications in later sessions.
